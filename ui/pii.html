The LLM boundary, verbatim

What actually reached the model

One card per question. Inside it, every payload that crossed the boundary, in order: the system prompt, the question, one row per tool result the agent fetched, and the model's reply at the bottom. Rows are not model calls; the agent answers one question over a few turns and each turn can carry several tool results.

Highlights mark pseudonyms. The badge says where the substitution happened, because the two approaches mask at different points. The agent masks tool output at this boundary, so its rows read masked here. The vanilla RAG corpus was already pseudonymized when it was serialized and embedded, so nothing is left to replace at call time and its rows read masked at ingest. Either way no real value reaches the model: account numbers, phone numbers, UPI IDs and loan references stay in the database. Click any payload to expand it.
